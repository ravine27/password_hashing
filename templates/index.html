What is Password Hashing?

Password hashing is the process of converting plaintext passwords into fixed-length strings using cryptographic algorithms. This irreversible transformation ensures that even if hackers access your database, they cannot retrieve original passwords.

üõ°Ô∏è

Why Hashing is Essential

Without hashing, plaintext passwords in your database create massive security risks. A single breach could expose all user credentials, leading to account takeovers across multiple services where users reuse passwords.

üßÇ Salting Explained

Adding random data to passwords before hashing prevents rainbow table attacks. Unique salts ensure identical passwords produce different hashes.

  • Auto: Algorithm generates and stores salt automatically
  • Manual: Developer must implement salt management
  • None: Extremely vulnerable to precomputed attacks

‚öôÔ∏è Adaptability Explained

Ability to adjust computational cost over time to counter hardware improvements and maintain security.

  • Memory & Time: Can adjust both memory usage and iteration count
  • Time Only: Can only adjust iteration count/work factor
  • Iterations: Only linear time adjustments possible

üß† Memory-Hardness Explained

Requires large, fast memory during computation, making GPU/ASIC attacks expensive and impractical.

  • Strong: 64+ MB per hash - defeats GPU clusters
  • Moderate: 16+ MB per hash - raises attack cost
  • No: Minimal memory - easy to parallelize

üéÆ GPU Resistance Explained

Resistance to parallel computation attacks using GPUs or custom hardware.

  • Excellent: Memory-hard design defeats parallel attacks
  • Good: Time-hard design slows parallel attacks
  • Poor: Easily parallelized on GPU hardware
